​   我认为，**异步**是一种编程模式，是用于对未来的状态进行提前定义处理方式的一种**编程手段**。

​    比如下面这段scala代码。

```scala
import scala.concurrent.ExecutionContext.Implicits.global
// 定义一个Future，并交给global executor执行
Future[String]{
    Thread.sleep(1000)
    "Hello"
}.foreach(println)
println("do something...")
Thread.sleep(2000)
// print: 
// do something...
// Hello
```

这里我们无需等待Future是否执行完成，直接进行下一步操作。
